Objectif

L’objectif n’est pas de “surveiller pour surveiller”, mais de convertir de l’information publique dispersée en compréhension collective, puis en propositions et redevabilité. Une veille citoyenne utile :

  • part de questions concrètes (budgets, marchés publics, urbanisme, santé, mobilité…) ;
  • s’appuie sur des faits vérifiables et une méthode explicitée ;
  • reste indépendante (ni parti, ni administration) ;
  • publie dans des formats accessibles, et ouvre la possibilité de corrections ;
  • relie l’analyse à des étapes d’action (droit de réponse, interpellation, suivi).
Ambition réaliste : passer d’un “dossier” à un réseau (3–4 structures pilotes), puis à une dynamique FWB (essaimage horizontal), en gardant une exigence : la qualité.

Sources & références

Pour rester robuste, la veille doit privilégier des sources primaires (documents officiels, budgets, décisions, textes, réponses administratives) et documenter les sources secondaires (presse, analyses, rapports). Règle simple : tout énoncé important doit pouvoir être retracé (lien + date + copie/archivage quand pertinent).

Références “socle”

  • Cadre EP FWB : analyse critique, participation active, émancipation, initiative démocratique et découverte des droits.
  • Veille citoyenne : suivi, analyse et influence de l’action publique en toute indépendance.
  • Contexte : complexité institutionnelle, défiance, désinformation, transparence.

Références “terrain”

  • Inclusion : langage clair, vulgarisation, formats accessibles (papier/infographies/vidéos) et complément présentiel.
  • Gouvernance : charte, pluralisme, validation collégiale, transparence des méthodes et des sources.
  • Professionnalisation : s’inspirer des méthodes d’IE pour structurer, sans copier la finalité (démocratie > compétition).
Mieux vaut 10 sources solides, bien utilisées, que 200 liens non digérés.

Cadre légal & éthique

Une veille citoyenne sérieuse se protège : elle évite la personnalisation, documente les faits, et met en place une procédure de correction. Cadre minimal :

  • Faits / interprétation : séparer constat et analyse.
  • RGPD : minimisation, anonymisation, prudence sur données sensibles.
  • Droit de réponse : canal + délai ; publication des réponses.
  • Traçabilité : conserver sources et méthode.
  • Collégialité : relecture avant publication sensible.

Récupération politique

Charte + pluralisme + règles anti-partisanes + validation collégiale.

Infobésité / intox

Tri, synthèses, pédagogie, outils de vérification, journal de corrections.

Fracture numérique

Présentiel + papier + formats multiples + médiation locale.

Pressions / résistances

Méthode, preuves, transparence, alliances, diplomatie.

Méthodes de veille (pas-à-pas)

  1. Périmètre : thème + territoire + question.
  2. Sourcing : sources primaires + 2–3 secondaires ; fréquence.
  3. Collecte : capturer (liens/PDF/captures), stocker, journaliser.
  4. Vérification : recoupement + outils de fact-checking ; expliciter l’incertitude.
  5. Analyse : faits → contexte → impacts → zones d’ombre → demandes.
  6. Publication : fiche + annexes ; formats accessibles.
  7. Interpellation : droit de réponse + questions précises ; relances.
  8. Suivi : mise à jour, corrections, versioning.
Qualité : résumé clair en tête + glossaire simple pour termes techniques.

Exemples de cas d’usage

  • Observatoire budgétaire communal : budgets, marchés, synthèses trimestrielles, questions au conseil communal.
  • Suivi d’engagements : tableau public promesses/plans → exécution.
  • Urbanisme : permis, enquêtes publiques, cartographie, chronologie.
  • Social/santé : suivi des mesures, accès aux documents, comparaison annonces/réalisations.

Plan d’action (6 semaines / 6 mois)

Option A — 6 semaines (prototype)

  1. S1 : cadrage + groupe + mini-charte + 3–5 sources.
  2. S2 : outillage + modèles + calendrier.
  3. S3 : collecte + archivage + glossaire.
  4. S4 : analyse + relecture + annexes.
  5. S5 : publication + droit de réponse + diffusion multiformat.
  6. S6 : interpellation + suivi + bilan + itération.

Option B — 6 mois (réseau pilote → essaimage)

  1. M1–2 : audit ; former 2–3 personnes ; repérer partenaires ; analyser axe EP.
  2. M3–4 : pilote local ; VOIR–PENSER–AGIR ; veille collaborative ; documenter.
  3. M5–6 : partenariats ; montée en outils ; financements selon ambition.
Structuration recommandée : cartographier acteurs/besoins → constituer un noyau pilote (3–4 organisations) → charte + gouvernance → production régulière → consolidation.

Indicateurs & évaluation

Participation

Nombre • diversité • rétention • couverture géo • participation hors-ligne

Production

Fiches • MAJ • sources primaires • glossaire • archives

Qualité

Recoupement • corrections • collégialité • RGPD • droit de réponse

Impact

Réponses institutions • changements • reprises médias • demandes citoyennes

Bon réflexe : co-définir 5–10 indicateurs avec les publics dès le départ.

Annexes

  • Glossaire (≤150 mots/terme) + index.
  • Modèles téléchargeables (.md/.pdf) : fiche, charte, communiqué.
  • Procédure d’errata + journal des versions.
  • Répertoire d’acteurs : associations, chercheurs, journalistes, juristes, data.
